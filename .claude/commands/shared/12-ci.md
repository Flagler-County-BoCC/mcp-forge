# Step 12 — Docker & CI/CD Pipeline

## Prerequisites
- Steps 0–11 applied.

## Applicability by projectType

| Artifact | http-api | library | cli | worker | mcp-server |
|---|---|---|---|---|---|
| Dockerfile | Required | Skip | Optional | Required | Optional |
| docker-compose.yml | Required | Skip | Skip | Required | Skip |
| GitHub Actions CI | Required | Required | Required | Required | Required |

**library:** No Docker. CI adds `npm publish --dry-run` on main pushes.
**cli:** Dockerfile optional (only if distributing as a Docker image). CI adds a smoke test (`node dist/index.js --help`).
**mcp-server:** Dockerfile optional (most users run directly via `node`). CI adds a smoke test. No `EXPOSE` and no HTTP `HEALTHCHECK`.

## Dockerfile — http-api, worker, mcp-server (optional)

Multi-stage build. For `mcp-server` and `worker`, omit `EXPOSE` and the HTTP `HEALTHCHECK`.

```dockerfile
FROM node:22-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev

FROM node:22-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY tsconfig*.json ./
COPY src ./src
RUN npm run build

FROM node:22-alpine AS production
ENV NODE_ENV=production
WORKDIR /app

RUN addgroup --system --gid 1001 nodejs \
  && adduser --system --uid 1001 appuser

COPY --from=deps /app/node_modules ./node_modules
COPY --from=build /app/dist ./dist
COPY package*.json ./

USER appuser

# http-api only: add EXPOSE and HTTP HEALTHCHECK
# EXPOSE <port>
# HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
#   CMD wget -qO- http://localhost:<port>/health/live || exit 1

CMD ["node", "dist/<entrypoint>.js"]
```

`<entrypoint>`: `server` (http-api) | `worker` (worker) | `stdio` (mcp-server) | `index` (cli)

Rules:
1. Never run as root (`USER appuser`).
2. Use `node:22-alpine` — not `node:latest`.
3. `npm ci --omit=dev` in deps stage.

## docker-compose.yml — http-api and worker only

```yaml
version: '3.9'
services:
  app:
    build: { context: ., target: production }
    ports: ["${PORT:-3000}:${PORT:-3000}"]   # http-api only
    env_file: [.env]
    depends_on: { db: { condition: service_healthy } }
    restart: unless-stopped

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: ${DB_USER:-postgres}
      POSTGRES_PASSWORD: ${DB_PASSWORD:-changeme}
      POSTGRES_DB: ${DB_NAME:-appdb}
    volumes: [postgres_data:/var/lib/postgresql/data]
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USER:-postgres}"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  postgres_data:
```

Omit `db` service if `detectedOrm === 'none'` and no DB in AUDIT_MANIFEST.

## GitHub Actions CI — .github/workflows/ci.yml

Select the variant matching `projectType`.

### http-api / worker

```yaml
name: CI
on:
  push: { branches: [main, develop] }
  pull_request: { branches: [main, develop] }
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci
      - run: npm run lint
      - run: npm run format:check
      - run: npm run typecheck
  test:
    runs-on: ubuntu-latest
    needs: quality
    services:
      postgres:
        image: postgres:16-alpine
        env: { POSTGRES_USER: postgres, POSTGRES_PASSWORD: testpassword, POSTGRES_DB: testdb }
        ports: ['5432:5432']
        options: --health-cmd pg_isready --health-interval 10s --health-timeout 5s --health-retries 5
    env: { NODE_ENV: test, DATABASE_URL: postgresql://postgres:testpassword@localhost:5432/testdb }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci
      - run: npm run test:coverage
      - uses: actions/upload-artifact@v4
        with: { name: coverage, path: coverage/ }
  build:
    runs-on: ubuntu-latest
    needs: test
    steps:
      - uses: actions/checkout@v4
      - uses: docker/setup-buildx-action@v3
      - uses: docker/build-push-action@v5
        with: { context: ., push: false, tags: app:ci, cache-from: type=gha, cache-to: 'type=gha,mode=max' }
```

Omit `services.postgres` block if no DB dependency.

### library

```yaml
name: CI
on:
  push: { branches: [main, develop] }
  pull_request: { branches: [main, develop] }
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
jobs:
  quality:
    runs-on: ubuntu-latest
    strategy:
      matrix: { node-version: ['22', '20'] }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '${{ matrix.node-version }}', cache: 'npm' }
      - run: npm ci
      - run: npm run lint
      - run: npm run format:check
      - run: npm run typecheck
  test:
    runs-on: ubuntu-latest
    needs: quality
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci
      - run: npm run test:coverage
      - uses: actions/upload-artifact@v4
        with: { name: coverage, path: coverage/ }
  publish-dry-run:
    runs-on: ubuntu-latest
    needs: test
    if: ${{ github.event_name == 'push' && github.ref == 'refs/heads/main' }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci
      - run: npm run build
      - run: npm publish --dry-run
```

### mcp-server

```yaml
name: CI
on:
  push: { branches: [main, develop] }
  pull_request: { branches: [main, develop] }
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci
      - run: npm run lint
      - run: npm run format:check
      - run: npm run typecheck
  test:
    runs-on: ubuntu-latest
    needs: quality
    env:
      NODE_ENV: test
      # CW_* vars for tests are mocked — no real credentials needed for unit tests
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci
      - run: npm run test:coverage
      - uses: actions/upload-artifact@v4
        with: { name: coverage, path: coverage/ }
  smoke-test:
    runs-on: ubuntu-latest
    needs: test
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci
      - run: npm run build
      # Send an empty JSON-RPC initialize message; server should respond and exit cleanly
      - run: |
          echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"ci-smoke","version":"0.0.0"}}}' \
          | timeout 5 node dist/stdio.js || true
```

## Determinism Rules
- Node version in CI must match `engines.node` (22).
- Postgres version in CI services must match `docker-compose.yml` (16-alpine).
- `concurrency.cancel-in-progress: true` is mandatory on all CI files.
- Do not add a deploy job.
- mcp-server smoke test uses `timeout 5` — the process is not expected to exit naturally; `|| true` prevents the step from failing on timeout.
